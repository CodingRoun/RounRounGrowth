# 文档体系标准与模板
> 本模板为 RounRounGrowth 工程开发文档体系标准
> 适用于单人模块化开发场景（M 阶段结构）
> 用于贯穿设计、实现与复盘全过程

---

## 一、简明流程图
```markdown
开发开始
   ↓
Mx-y.md          ← 模块设计与进度（开发前）
   ↓
代码实现
   ↓
Mx-yWrapUp.md    ← 阶段总结与经验（开发后）
   ↓
CHANGELOG.md     ← 全局项目版本记录

```

---

## 二、Mx-y.md（模块设计与进度文档）
> 阶段设计规范与进度对照表
> 每个模块开发前填写，开发中可持续补充

```markdown
# Mx-y

## 一、模块目的
> 简要说明模块在系统中的职责、功能定位与设计目标

---

| 功能ID | 分类 | 功能描述 | 状态 (已完成/正在进行/延后/丢弃) |
|-------|-------|---------|------------------------------|
| Mx-y-F01 | 核心 |  |  |
| Mx-y-U01 | 扩展 |  |  |

**命名规则说明：**
| 类型 | 命名规则 |
|------|---------|

| 测试点 | Mx-y-T01 |

## 三、数据结构约定
> 类型定义：记录类 / 结构 / 枚举的基础信息  
> 字段定义：记录其成员字段及外部引用

**类型定义**
| 定义文件 | 类型名 | 类型类别 | 描述 | 枚举成员（仅 enum 时） |
|---------|-------|---------|------|----------------------|
| .cs |  | class |  | - |
| .cs |  | enum |  | [ValueA, ValueB, ...] |

**字段定义**
| 所属类型 | 字段名 | 数据类型 | 访问级别 | 特性 | 说明 |
|---------|-------|---------|---------|------|-----|

---

## 四、模块接口
| 所属类型 | 方法名 | 输入 | 输出 | 行为描述 |
|---------|-------|------|-----|---------|

---

## 五、模块间关系
| 调用方 | ↔ | 被调用方 | 调用时机 | 调用方法 | 说明 |
|-------|---|---------|---------|---------|------|

**关系符号说明：**
| 符号 | 含义 |
|------|------|
| → | 调用方依赖被调用方 |
| ← | 被调用方反向通知调用方（回调/事件） |
| ∥ | 平行模块，无直接依赖，仅资源共享 |

---

## 六、核心交互与状态流转（如适用）

|-----|----------|---------|---------|

**状态机时序**

→ 否 → 执行1

↓  
用户操作 → 系统响应
```
