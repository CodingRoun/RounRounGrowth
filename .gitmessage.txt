# ============================================
# RounRoun Growth Commit Message Template
# ============================================
#
# 提交结构：
# <type>: [Mx-y] 标题（番茄#n）
# <空行>
# <详细说明（可选）>
# 子动作 A；
# 子动作 B；
# （可选）验证情况。
#
# --------------------------------------------
# 类型说明（type）：
# --------------------------------------------
# feat:     新功能（Feature）—— 实现一个番茄任务的功能代码
# fix:      修复（Bug Fix）—— 修复错误或异常行为
# refactor: 重构（Refactor）—— 改进代码结构，不改变功能
# chore:    杂项（Chore）—— 重命名、注释、配置、日志
# style:    样式（Style）—— UI或排版调整，不涉及逻辑
# docs:     文档（Docs）—— 注释、字典卡、文档更新
# test:     测试（Test）—— 测试逻辑或验证脚本

# --------------------------------------------
# 示例：
# --------------------------------------------

# feat: [M0-5] 实现 SwipeToNavigate 页面切换（番茄#6）
#
# 将 PageSwipeDetector 的 OnSwipe 事件与导航器绑定；
# 实现左右滑动页面切换；
# 支持边界 return。

# fix: [M0-5] 修复边界滑动不触发提示的逻辑（番茄#8）
#
# 调整 BuildingNavigationTable.GetNextRoomId 的边界判断。

# chore: [M0-5→6] 日终提交

# --------------------------------------------
# 提交规范：
# --------------------------------------------
# 1. 每次提交只做一件可验证的事（建议对应一个番茄任务）。
# 2. 第一行简洁描述（不超过 50 字）。
# 3. 第二行空一行后写详细说明（每行不超过 72 字）。
# 4. 每日收工或阶段结束可做汇总提交（chore）。
# 5. commit.message 会作为 changelog 来源，请保持一致性。
# --------------------------------------------
